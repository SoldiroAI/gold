Core Mechanics

Each market has YES and NO tokens. Holding 1 YES + 1 NO always equals $1.00 USDC at resolution. You can buy YES at $0.30 if you believe an event has >30% probability — if it resolves YES, you profit $0.70 per share.

Order Types

Polymarket operates a hybrid CLOB: Limit orders (maker, 0% fee, add liquidity) and Market orders (taker, ~1-2% fee, instant fill). For automation, prefer limit orders near the mid-price to capture the spread.

Token Math

YES price + NO price ≈ $1.00 (minus spread). If YES = $0.65, NO = $0.35. Mispriced markets exist when YES + NO ≠ $1.00, creating arbitrage. Resolution is handled by UMA's Optimistic Oracle on-chain.

Liquidity Reality

High-volume markets (>$500K) have tight 1-2¢ spreads. Low-volume markets (<$50K) can have 5-15¢ spreads. Your $200 capital is safest in markets with >$100K volume to avoid moving the price and getting filled at bad levels.

Smart Money — Wallet Analysis Framework

All Polymarket trades are on-chain (Polygon). Every wallet's full history is publicly visible. The goal is to identify wallets with repeatable edges — not lucky one-time wins.

💡
Key Insight: Only ~16.8% of wallets on Polymarket show net gains. Focus on wallets with 55%+ win rates over 50+ trades. High P&L alone is misleading — it can reflect one concentrated political bet.

Copy-Trade Signal Rules

1

Consensus Filter

Only act when 2–3 tracked whale wallets take the SAME position within a short window (15–60 min). Single whale entries can be noise or hedging.

2

Price Slippage Check

If the current price has moved >10% from the whale's entry, skip the trade. Your edge has likely already disappeared — the market repriced the information.

3

Liquidity Guard

Never enter markets with <$100K total volume when copy-trading. Thin markets mean your order itself moves the price unfavorably.

4

Independent Exit

Set your own profit target (1.5–2.5x) and stop-loss (-30% of position). Don't wait to see if the whale exits — they have much more capital tolerance.

Three Core Strategies

STRATEGY A — Whale Copy with Lag Filter

How it works: Monitor 5–8 pre-screened high-win-rate wallets via the Data API. When 2+ wallets buy the same token within 30 minutes, AND current price is within 8% of their entry, enter a proportional position.


Position Size: Max $20 per trade (10% of capital). Scale to $30 only if 3+ whales confirm.


Exit: Sell at +50% gain OR -30% loss OR when 1 confirming whale exits, whichever comes first.


Best for: Liquid markets >$200K Entry delay: 5-15 min after whale
STRATEGY B — Overreaction Fade

How it works: When a market price moves >15% in <2 hours due to a news event, look for over/underreaction. If YES shoots from 0.30 → 0.55 on ambiguous news, and whale wallets are NOT following, fade the move by buying NO (which dropped from 0.70 → 0.45).


Signal: Price delta >15% in 2h + low whale participation + thin order book above current price.


Position Size: Max $15 per trade. Higher uncertainty requires smaller size.


Best for: Crypto & finance markets Hold: 1-48 hours
STRATEGY C — Cross-Platform Arbitrage (Polymarket vs Kalshi)

How it works: Same event priced differently on Polymarket vs Kalshi. Buy YES on the cheaper platform, NO on the more expensive one. Lock in guaranteed profit regardless of outcome.


Example: "Bitcoin >$95K" = 45¢ on Polymarket, 52¢ on Kalshi. Buy YES on Polymarket + NO on Kalshi. Net expected profit = 3-5¢ per dollar deployed after fees.


Constraint: Must account for taker fees (~1-2%), capital lock-up time, and gas costs on Polygon. Only viable when spread >4¢.


Best for: High-volume markets Risk: Capital lockup
Section 04
Automation Logic — Complete Flow
🔧
Implementation Note: You'll need Python 3.10+, the official py-clob-client or polymarket-apis PyPI package, a Polygon wallet with USDC, and an approved proxy wallet on Polymarket. API keys are generated in the Polymarket UI under Settings → API.

Master Bot Architecture

1

SCANNER — Market Discovery Loop (every 5 min)

Pull active markets from Gamma API. Filter by: volume >$100K, not resolving within 24h, spread <5¢. Build a watchlist of 10-20 qualifying markets.

2

WHALE MONITOR — Wallet Tracker (every 2 min)

Poll Data API for new trades from your pre-screened wallet list. Detect when 2+ wallets take the same side in a watchlist market within a rolling 30-min window.

3

SIGNAL VALIDATOR — Entry Filter

Check: (a) Current price within 8% of whale entry. (b) Order book has enough depth to absorb your size without moving price >2¢. (c) Daily loss limit not breached. (d) Max open positions ≤ 5.

4

ORDER EXECUTOR — Place Limit Order

Place limit order at mid-price or 1¢ below ask (to be filled as maker, saving fees). Size = min($20, 10% of remaining capital). Log the trade with entry price, timestamp, triggering wallets.

5

POSITION MONITOR — Exit Logic (every 10 min)

For each open position: check current price vs. entry. If +50% → place limit sell. If -30% → place market sell (accept taker fee). If whale exits → flag for manual review or auto-exit.

PYTHON · bot_engine.py
import time, logging from polymarket_apis import GammaClient, DataClient, ClobClient from config import WALLET_ADDRESS, API_KEY, WATCHED_WALLETS # ───────────────────────────────────────────── # CONFIG — Conservative $200 parameters # ───────────────────────────────────────────── CONFIG = { "total_capital": 200.0, # USDC "max_position_pct": 0.10, # 10% per trade = $20 max "max_open_positions": 5, "min_market_volume": 100_000, # $100K minimum "max_spread": 0.05, # 5¢ max spread "whale_window_min": 30, # consensus window "whale_consensus": 2, # min 2 wallets agree "max_price_lag_pct": 0.08, # skip if price moved >8% "take_profit_pct": 0.50, # +50% → sell "stop_loss_pct": -0.30, # -30% → cut loss "daily_loss_limit": 20.0, # max $20 loss per day } # ───────────────────────────────────────────── # SCANNER — find qualifying markets # ───────────────────────────────────────────── def scan_markets(gamma: GammaClient) -> list: markets = gamma.get_markets(active=True, limit=100) watchlist = [] for m in markets: if (m.volume >= CONFIG["min_market_volume"] and m.spread <= CONFIG["max_spread"] and m.hours_to_close > 24): watchlist.append(m) return watchlist # ───────────────────────────────────────────── # WHALE MONITOR — detect consensus trades # ───────────────────────────────────────────── def check_whale_consensus( data: DataClient, watchlist: list, recent_trades: dict ) -> list: signals = [] for wallet in WATCHED_WALLETS: trades = data.get_trades(user=wallet, window_min=30) for trade in trades: if trade.market_id in [m.id for m in watchlist]: key = (trade.market_id, trade.side) # (market, YES/NO) recent_trades[key].append({ "wallet": wallet, "price": trade.price, "time": trade.timestamp }) # Emit signal if consensus threshold met for key, entries in recent_trades.items(): unique_wallets = set(e["wallet"] for e in entries) if len(unique_wallets) >= CONFIG["whale_consensus"]: avg_entry = sum(e["price"] for e in entries) / len(entries) signals.append({"market_id": key[0], "side": key[1], "avg_whale_entry": avg_entry, "confirming_wallets": unique_wallets}) return signals # ───────────────────────────────────────────── # SIGNAL VALIDATOR — gate before execution # ───────────────────────────────────────────── def validate_signal(signal, clob: ClobClient, portfolio) -> bool: book = clob.get_order_book(signal["market_id"]) current_price = book.mid_price # Gate 1: price hasn't moved too far lag = abs(current_price - signal["avg_whale_entry"]) / signal["avg_whale_entry"] if lag > CONFIG["max_price_lag_pct"]: return False # Gate 2: not too many open positions if len(portfolio.open_positions) >= CONFIG["max_open_positions"]: return False # Gate 3: daily loss limit if portfolio.daily_loss >= CONFIG["daily_loss_limit"]: return False # Gate 4: order book depth (don't move price >2¢) position_usdc = CONFIG["total_capital"] * CONFIG["max_position_pct"] depth_ok = book.can_absorb(side=signal["side"], usdc=position_usdc, max_slippage=0.02) return depth_ok # ───────────────────────────────────────────── # ORDER EXECUTOR — place limit order # ───────────────────────────────────────────── def place_entry(signal, clob: ClobClient, portfolio) -> dict: book = clob.get_order_book(signal["market_id"]) limit_price = round(book.best_ask - 0.01, 2) # 1¢ below ask = maker order size_usdc = CONFIG["total_capital"] * CONFIG["max_position_pct"] shares = size_usdc / limit_price order = clob.create_limit_order( token_id=signal["market_id"], side=signal["side"], price=limit_price, size=shares, ) portfolio.add_position({ "order_id": order.id, "entry_price": limit_price, "shares": shares, "stop_loss": limit_price * (1 + CONFIG["stop_loss_pct"]), "take_profit": limit_price * (1 + CONFIG["take_profit_pct"]), }) logging.info(f"ORDER PLACED: {signal['side']} {shares:.1f} shares @ {limit_price}") return order # ───────────────────────────────────────────── # MAIN LOOP # ───────────────────────────────────────────── def run_bot(): gamma, data, clob = GammaClient(), DataClient(), ClobClient(API_KEY) portfolio = Portfolio() recent_trades = defaultdict(list) while True: watchlist = scan_markets(gamma) signals = check_whale_consensus(data, watchlist, recent_trades) for signal in signals: if validate_signal(signal, clob, portfolio): place_entry(signal, clob, portfolio) portfolio.monitor_exits(clob, CONFIG) # check TP/SL time.sleep(120) # scan every 2 minutes
